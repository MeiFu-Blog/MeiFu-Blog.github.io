前言

lvs 是 Linux Virtual Server 的简写，意即 Linux 虚拟服务器，是一个虚拟的服务器集群系统。本项目在 1998 年 5 月由章文嵩博士成立，是中国国内最早出现的自由软件项目之一。目前 lvs 已经被集成到 Linux 内核模块中。Keepalived 是 Linux 下一个轻量级别的高可用解决方案。

学习目标

◆ 掌握集群的分类、作用以及集群在生产环境中的适用场景
◆ 掌握 lvs、nginx 等负载均衡工具的配置与使用
◆ 掌握高可用集群 keepalived 的配置与使用

掌握内容

集群介绍：集群作用，集群分类、适用场景，负载均衡分类

1、集群是指将多台计算机联合起来，作为一个整体来使用，以提高计算机系统的可靠性、可用性、可扩展性和负载均衡能力。

2、常见的集群分类有负载均衡集群、高可用性集群、高性能计算集群和网格计算集群。
其中，负载均衡集群是互联网行业常用的集群架构模式之一，它可以把一个高负荷的应用分散到多个节点共同完成，
适用于业务繁忙、大负荷访问的应用系统

3。负载均衡分类包括硬件负载均衡和软件负载均衡。

4、适用场景包括：
业务繁忙、大负荷访问的应用系统
高性能计算
大数据处理
网络存储

lvs 四层负载均衡：lvs 介绍，lvs 优势与不足，lvs 核心组件

1、LVS是Linux Virtual Server的简称，是一个由章文嵩博士发起的自由软件项目，它的官方站点是www.linuxvirtualserver.org。
LVS是一个高性能、高可用性和可伸缩性的服务器集群软件，主要用于构建高性能、高可用性和可伸缩性的Web服务集群。

2、LVS的优势包括：
高并发连接：LVS基于内核网络层面工作，有超强的承载能力和并发处理能力。单台LVS负载均衡器，可支持上万并发连接。
稳定性强：是工作在网络4层之上仅作分发之用，这个特点也决定了它在负载均衡软件里的性能最强，稳定性最好，对内存和CPU资源消耗最少。
可扩展性好：LVS可以通过增加负载均衡器、后端服务器等方式来扩展集群规模。
LVS的不足包括：
LVS只能做四层负载均衡，不能做七层负载均衡。
LVS只能对IP地址和端口进行负载均衡，不能对HTTP头进行负载均衡。

3、LVS的核心组件包括：
虚拟服务器
负载调度器
后端真实服务器

lvs 负载均衡工作模式

LVS有四种工作模式：
NAT模式（LVS-NAT）
直接路由 (Direct Routing)模式（LVS-DR）
IP隧道 (Tunnel)模式（LVS-TUN、不常用）
FULL-NAT模式（双向转换模式、不常用）

ipvsadm、持久连接、调度算法、NAT 模式部署、DR 模式部署

1、ipvsadm是LVS在应用层的管理命令，可以通过这个命令去管理LVS的配置。

2、持久连接是指客户端与服务器之间的连接在一段时间内保持不变，不会因为客户端和服务器之间的交互而断开。

3、调度算法有静态方法和动态方法。静态方法仅根据算法本身进行调度，如轮询、加权轮询、源IP地址hash等。
动态方法则考虑各RS当前的负载状态，如加权最小连接数、加权最少请求、源IP地址hash等。

4、NAT模式部署和DR模式部署是LVS负载均衡群集部署的两种方式。
NAT模式下，LVS作为一个NAT设备，将客户端请求转发到后端真实服务器上；
这种部署方式可以隐藏后端真实服务器的IP地址，提高了安全性，但是会增加一定的延迟和额外的网络开销。

DR模式下，LVS作为一个路由器，将客户端请求转发到后端真实服务器上。
这种部署方式可以减少网络开销和延迟，但是需要在后端真实服务器上配置额外的路由规则，同时也暴露了后端真实服务器的IP地址。

Nginx 七层负载均衡：介绍、配置，算法（轮询、ip_hash、fair 、url_hash）

介绍：Nginx是一款高性能的Web服务器，也是一款高性能的反向代理服务器。Nginx支持多种负载均衡算法，包括轮询、ip_hash、fair、url_hash等1。

配置：在Nginx中，可以通过配置upstream模块来实现七层负载均衡。

1、轮询算法是默认的负载均衡算法，将请求轮流分配给后端服务器，每个服务器接收到的请求数量大致相等。
2、ip_hash算法则是根据客户端的IP地址进行哈希计算，同一个IP客户端固定访问一个后端服务器，可以解决session问题；
3、fair算法则是根据后端服务器的响应时间进行分配；将请求分配给响应时间最短的服务器，可以提高系统的性能和可靠性。
4、url_hash算法则是根据请求的URL进行哈希计算，将同一URL的请求分配给同一台后端服务器，可以提高缓存的效率。

七层负载均衡项目实战：Tomcat 多实例部署，Tomcat 负载均衡集群

Tomcat是一个常用的Java应用服务器，可以通过多实例部署和负载均衡集群来提高应用的可用性和性能。下面介绍Tomcat多实例部署和Tomcat负载均衡集群的实现方法。

一、Tomcat多实例部署

创建Tomcat实例
在Tomcat的安装目录下，复制一份Tomcat的目录，例如复制一份Tomcat8.5目录，重命名为Tomcat8.5-1，表示第一个Tomcat实例。

修改Tomcat实例配置
进入Tomcat8.5-1目录，修改conf/server.xml文件，将端口号修改为不同的端口号，例如将HTTP端口号修改为8081，将AJP端口号修改为8001。

启动Tomcat实例
进入Tomcat8.5-1目录，执行bin/startup.sh（或startup.bat），启动Tomcat实例。

创建多个Tomcat实例
重复上述步骤，创建多个Tomcat实例。
